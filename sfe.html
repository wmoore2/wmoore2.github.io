Introduction

Solutions for Education (SFE) is a startup created in 2021 by Paul Moore. The company aims develop cloud-based software to aid educators in various ways. The most notable being a website dedicated to automating the generation of report card comments. In my time working at SFE, I did full stack development and began writing the product from scratch. I learned an immense amount about the tech stack used in this time.

Goals

My goals for this work term were largely about learning new things, most notably learning new programming languages and frameworks.

I set myself a goal to learn two new languages: Python and TypeScript. I was successful in this goal, as I became proficient in writing Python (using the Django library with Django RESTful API). I also became proficient at writing TypeScript, a statically type checked superset of the JavaScript programming language. In learning TypeScript, I also learned about Babel: a transpiler, and Webpack: a JavaScript bundling package.

Another technology I wanted to learn about was the frontend TypeScript framework React. I successfully learned how to write React with TypeScript and was able to create a web application using primarily this framework. I found React easy to pick up with plenty of more advanced tools to use once I became more familiar with its constructs.

Finally, the last goal I had was to deploy an application. I decided to use Heroku for deployment, as it is free, scalable, and easy to get started with. I successfully learned to use Heroku in its basics and was able to deploy my web application through it.

Job Description

My role at SFE consisted of entirely full stack developer duties. I was responsible for everything from authentication to frontend.

I wrote the website from nothing, using Django on the backend and React with TypeScript on the frontend. Writing authentication was a large part of this project. I wanted to learn how to write a simple authentication system which would allow users to create accounts and login/logout of them. I accomplished this with the use of JSON Web Tokens in conjunction with an addon for Django RESTful API to generate the tokens on the backend for valid users and store them on the frontend as cookies. This took the most time, learning all the technologies involved and I feel as if I got a lot out of the experience.

Brainstorming the architecture of how user input data would be stored was another big part of the position. I was responsible for creating every Django model used. This includes a customized user model for use with authentication, as well as models for every unique piece of data the website would handle. This includes user created classes (as in school classes), subjects, students, and marks. This involved figuring out how to store each piece of data in relation to others, as the data is more complicated than meets the eye. The approach I took is faster and more scalable than using a non-relational database and storing plain JSON or something similar. I used a relational PostgreSQL database which interfaced quite well with Django. Each model would have a foreign key relation to another model as well as the user which created the object, to easily retrieve all objects associated with a specific user.

There is much more planned for the website, I was only able to complete the mark book portion, which allows teachers to enter student information. This is vital for the eventual generator. The next step is to plan the generator out fully and implement it.

Conclusions

This work term gave me a lot of freedom to use the technologies I wanted to, work at my own pace, and build a website from scratch. I could keep talking about my experience, but I have decided to keep it somewhat concise. I learned a great deal about Python, Django, React, TypeScript, and full stack development in general. Overall, this experience was a great one and my summer was certainly put to good use, constantly learning new things, and honing my abilities as a programmer.
